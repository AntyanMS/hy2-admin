# Каскадное подключение (черновик для mskgw / exit)

Панель управляет **Hysteria2** и при необходимости читает **sing-box** (`PANEL_BACKEND=sing-box` в `.env`). Каскад «клиент → средний узел → выход» сводится к двум отдельным задачам на разных хостах; автоматизация по SSH зависит от ваших ключей и политики доступа к **mskgw**.

## Роли узлов

1. **Exit** — сервер с HY2 Admin, `hysteria-server` с публичным адресом клиентов.
2. **Relay / mskgw** — узел с ограниченной политикой маршрутизации (например sing-box + reality/VLESS к выходу). В репозитории есть вспомогательные скрипты в `tools/` (пути и секреты нужно подставить под ваш инфраструктурный контур):
   - `tools/mskgw_route_ru_domains.py` — идея маршрутизации доменов;
   - `tools/fix_singbox_to_msgw_network.py` — правки JSON sing-box под связку с msgw;
   - примеры nginx: `tools/msgw.mooo.com.nginx.conf`, заглушки в `tools/msgw-https-root-stub/`.

## Рекомендуемый порядок

1. Развернуть **exit** обычным установщиком (`install_hy2_admin.sh`), выдать пользователям конфиги.
2. На **mskgw** настроить sing-box/Hysteria по документации проекта (отдельно от панели): upstream на exit, sniff/route по вашей схеме.
3. Включить на mskgw в `.env` панели (если панель стоит и там): `PANEL_BACKEND=sing-box`, `SING_BOX_CONFIG_PATH=/etc/sing-box/config.json` — только мониторинг/часть UI, без «магии» каскада в одном клике.

## Автоматизация

Полноценный сценарий «одна команда с локальной машины → SSH на mskgw» не закреплён в репозитории: нужны host, user, неинтерактивный `sudo`, версия sing-box. Рекомендуется оформить это в вашем Ansible/скрипте с секретами вне git.

Дальнейшая интеграция (роль `relay`, пресеты outbound) может быть добавлена отдельным PR после фиксации целевой топологии (только sing-box, только HY relay, или смесь).
