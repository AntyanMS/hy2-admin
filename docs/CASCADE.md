# Каскадная схема (черновик)

## Роли

- **Gateway node** — входной узел, где клиенты подключаются к HY2.
- **Exit node** — удалённый узел для транзита non-local трафика.
- **Panel node** — узел с веб-панелью управления (может совпадать с gateway).

## Базовая идея

1. Пользователи и лимиты управляются на panel/gateway узле.
2. Данные пользователей синхронизируются на exit узел.
3. Локальный трафик идёт напрямую, внешний — через exit.

## Что должно быть в установке

- Генерация fingerprint/token для exit-узла.
- Регистрация exit-узла на master-панели.
- Автосинхронизация пользователей с подписью запросов.
- Проверка состояния сервисов и журналов.
